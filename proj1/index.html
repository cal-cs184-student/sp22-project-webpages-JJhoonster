Section I: Rasterization

Part 1: Rasterizing single-color triangles

So I rasterized a single-color triangle by using the Three Line Point-in-Triange Test from lecture. I first established sample bounds defined by min/max x/y coordinates. I then sampled every pixel within the bounds and checked if it was inside the triangle using line equation tests. If the results of all three line equation tests had the same sign, that meant the sample was inside the triangle. Samples that were determined to be inside the triangle had their pixels filled in. This algorithm is no worse than one that checks each sample within the bounding box of the triangle because it does exactly that.

Part 2: Antialiasing triangles

Supersampling is useful for anti-aliasing and getting rid of jaggies/hard edges. My supersampling algorithm was very similar to my initial single-color rasterizer algorithm. I actually didn't touch any function other than rasterize_triangle in the end, because I couldn't get them to work. Instead, I did color math and determined pixel color based on the percentage of samples inside the triangle. Apparently this assumes that the outside of the triangle is white, so it doesn't really work well, but I couldn't figure it out the other way.
